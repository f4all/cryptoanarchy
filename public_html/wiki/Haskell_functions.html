[edit] Functions

Functions are unable to take initiatives, think of them as just filters for data. Something else has to pour the data through these filters. Monads seems to be the stuff that takes the initiatives and make stuff happen.

Functions (and all other code) generally use recursion rather than loops for repetitive tasks, such as traversing lists. Functions are nearly always defined in themselves.

[edit] Learning to create haskell functions

Learning this is backwards and strange. The idea of the imperative programming paradigm, that describes computation in terms of statements that change a program state, needs to be complemented with the functional paradigm. This is not so very simple and perhaps requires some practice.

[edit] Example function 

-- 
-- returns the running sum of a list. [1,2,4,9] becomes [1, (1+2), (1+2+4), (1+2+4+9)] = [1,3,7,16]
--
scanSum :: [Integer] -> [Integer]
scanSum = scanSum' 0

scanSum' :: Integer -> [Integer] -> [Integer]
scanSum' tot [] = []
scanSum' tot (x:xs) = x + tot : scanSum' (x + tot) xs

What does this mean?

  • The green code is the type of the functions.
    • ScanSum takes a list of Integers and turns them into a list of Integers.
    • ScanSum' takes an Integer, a list of Integers and turns both of them into a list of Integers.
  • scanSum is defined as scanSum' ("scanSum prime") where the first argument is zero. (This is called currying)
  • scanSum' uses the argument tot just as a function in an imperative language (such as C/Java) would use a local variable.
  • scanSum' is defined on two rows.
    • The first line says: When the list of Integers is empty, return a empty list.
    • The second line says: Otherwise, the first element in the new list is equal to tot + the first element in the list, concatenated with the result of scanSum' when applied to the remaining list. The computation will end with the empty list, which is described in the line above.

It is possible to define a new dataset consisting of scanSum applied over the infinite set of natural numbers (1,2,3,4..), and then at some later time have some function collect the first 10 elements from this new set. Only for the datapoints that needs evaluation are computations then made.

The alien Mr. Spock would have loved this.

[edit] Some random functions 

--
-- multiplies a list of integers with a constant
--
multiplyList :: Integer -> [Integer] -> [Integer]
multiplyList _ [] = []
multiplyList m (n:ns) = (m*n) : multiplyList m ns



--
-- returns the first n elements from a list
--
takeInt :: Integer -> [a] -> [a]
takeInt 0 _          = []
takeInt _ []         = []
takeInt n (x:xs)     = x : takeInt (n-1) xs


--
-- removes the first n elements from a list
--
dropInt :: Integer -> [a] -> [a]
dropInt 0 bs = bs
dropInt a (b:bs) = dropInt (a-1) bs
dropInt _ _ = []


--
-- alternative of dropInt that does the very same thing.
--
dropInt2 :: Integer -> [a] -> [a]
dropInt2 0 list     = list
dropInt2 _ []       = []
dropInt2 n (x:xs)   = dropInt (n-1) xs


-- 
-- returns the sum of all integers in a list
--
sumInt :: [Integer] -> Integer
sumInt (x:xs) = x + sumInt xs
sumInt [] = 0


-- 
-- returns the running sum of a list. [1,2,4,9] becomes [1,3,7,16]
--
scanSum :: [Int] -> [Int]
scanSum = scanSum' 0
 
scanSum' :: Int -> [Int] -> [Int]
scanSum' tot [] = []
scanSum' tot (x:xs) = x + tot : scanSum' (x + tot) xs


--
-- creates a new list that consists of the difference between all integers in the list. [1,3,9] becomes [(3-1),(9-3)] = [2,6]
--
diffs          :: [Integer] -> [Integer]
diffs []       = []
diffs (x:[])   = []
diffs (x:y:xs) = (y-x) : diffs (y:xs)
