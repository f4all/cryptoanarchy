[edit] nice functions

random nice functions that are defined in the prelude.

[edit] Lists

  • head -- returns the first object of a list
  • last -- returns the last object of a list
  • tail -- returns all except the first object of a list
  • init -- returns all except the last object of a list
  • (!!) -- returns the n'th object of a list. example: [1..10] !! 3 == 4
  • (++) -- concatenates two lists. example [1..3] ++ [4..6] == [1..6]

[edit] Processing lists

All of these applies functions on all objects in a list, to turn the list into something else.

  • map :: (a -> b) -> [a] -> [b]
  • foldr :: (a -> b -> b) -> b -> [a] -> b -- a:b:c:[] -> f a (f b (f c z))).
    • Takes a function of type (a -> b -> b), such as (+), and applies it to all objects in a list, resulting in a single object. Example: Sum of all integers in a list.
  • foldl :: (a -> b -> b) -> b -> [a] -> b -- a:b:c:[] -> f (f (f z a) b) c. memory intensive, quicker than foldr (depends). not to be used at infinite lists.
    • folds can also be used to create lists out of list of list. Think of it as the derivate function, but for lists ;)
  • foldl' is the same as foldl but computes the answer immediately (strictly). import Data.list
  • foldr1 and foldl1 are like foldr and foldl but simpler.
  • filter :: (a -> Bool) -> [a] -> [a] -- removes all items for which the function (a -> Bool) returns false.

[edit] Strings

  • read -- converts a String to Integer or Double. (reads the content of a string and turns it into something.)

[edit] Left/right-associativity

  • f(f(f a b) c) d is left-associative. It performs calculations from left to the right. (foldl)
  • f a (f b (f c d))) is right-associative. It performs calculations from right to the left. (foldr)

Functions that when unwrapped looks like f(f(f(f...) are optimized into loops by the compiler, which makes them faster. However, f is placed in a list of computations to be performed in the future, which makes it kind of memory-intensive. Usage of strictness can counter this problem, by forcing the compiler to generate code that immediately computes.

foldl' is strict and is defined in Data.list. Use it instead of foldl.
